# 規劃紀錄（Plans）

這裡保存本專案每次重大變更前的**規劃文件**。

Git commit 記錄的是「做了什麼」，這些計畫記錄的是「**為什麼這樣做、當時考慮過哪些選項、跟使用者確認了什麼**」——那些資訊在 commit message 裡通常留不下來。

## 為什麼要複製到這裡

Claude Code 的計畫檔預設存放在 `~/.claude/plans/`，那個位置有三個問題：

1. **不在版控裡** — `~/.claude/` 不是 git repo，改掉就沒有歷史。
2. **不在專案裡** — 換電腦、重灌或誤刪家目錄設定就一起消失，也不會跟著 repo 走。
3. **同一個對話會覆寫同一個檔案** — 後續重新規劃時是整份改寫，不是新增。

第 3 點實際發生過：`01-single-page-tutorial.md` 的原始檔案已經被第二次規劃覆寫掉了，這份是**從當時的對話記錄還原**的核准版本（內容與核准時一致，僅在開頭加註說明）。

## 慣例

日後若再有重大變更，建議在計畫核准後就複製一份進來：

```bash
cp ~/.claude/plans/<該對話的計畫檔>.md docs/plans/0X-<簡短描述>.md
```

檔名用**遞增編號 + 英文簡述**（`~/.claude/plans/` 的原始檔名是自動產生的亂數字串，沒有意義，不建議沿用）。
