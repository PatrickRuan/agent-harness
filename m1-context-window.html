Module 1 · 全課的地基

那個箱子
模型其實沒有記憶

這一課只講一件事。但這件事撐起後面所有的課——包括那些聽起來很潮的新名詞。

1先講一個違反直覺的事實

你跟 ChatGPT 聊了半小時，它好像記得你剛剛說過什麼。這是幻覺。

大型語言模型（LLM）本身沒有任何記憶。它不會「記住」你上一句話，就像計算機不會記得你上一次按了什麼。

每一次呼叫模型，對它來說都是人生第一次。它看到一堆文字，然後預測接下來最合理的文字是什麼。如此而已。

那它為什麼看起來記得？因為外面那支程式，每一輪都把整段對話重新寄一次給它。你以為它記得，其實是有人每次都把前情提要重講一遍給一個失憶的人聽。

2那個箱子有多大？

既然每輪都要把全部重寄，就有一個殘酷的物理限制：一次能寄多少？

模型一次能讀進去的文字量是有上限的。這個上限叫做脈絡視窗（context window），單位是 token（你可以粗略想成「字」，中文大約 1 個字 ≈ 1～2 個 token）。

這個箱子放得下多少，就決定了模型「當下能同時看到多少東西」。超過的部分，它看不到——不是忘記，是根本沒進去。

給電機系的類比：Karpathy（前 Tesla AI 總監）自己的比喻是——模型是 CPU，脈絡視窗是 RAM。 CPU 再強，能算的也只有「這一刻載進 RAM 裡的東西」。硬碟裡的資料再多，沒載進來就等於不存在。 後面整門課，其實都在講同一件事：誰、在什麼時候、決定把什麼載進這塊 RAM。

箱子滿了會怎樣？

不會有酷炫的爆炸。會發生的是三件很煩的事：

  • 直接被拒絕 — 請求太長，模型根本不收。
  • 前面的被丟掉 — 為了塞下新的，只好把最早的對話砍掉，於是它「忘了」你一開始的要求。
  • 被壓縮（compaction） — 程式把前面的對話濃縮成一段摘要再繼續。摘要一定會失真，細節就掉了。

你如果用過 AI 寫程式工具，聊久了它開始鬼打牆、忘記你三十分鐘前訂的規則——十之八九就是這個箱子在作怪。

4為什麼這是全課的地基

接下來幾課會出現一堆名詞：context engineering、harness、sub-agent、MCP、Skills、multi-agent……乍看很多，其實全部都是同一個問題的不同解法：

名詞翻成「箱子」的話就是……
Prompt engineering把箱子裡的字寫好
Context engineering決定箱子裡該放什麼
Harness負責裝箱、並反覆呼叫模型的那支程式
Loop engineering箱子每輪都要重裝，設計這個循環
Sub-agent（子代理）另外開第二個箱子，別把主箱子弄髒
MCP往箱子裡插工具的標準插頭
Skills說明書需要時才載入箱子，不用時不佔空間
RAG（檢索增強）去外面撈對的資料放進箱子

所以如果你這一課只帶走一句話，帶走這句：

模型沒有記憶，而它的視野是一個有限的箱子。
剩下的一切，都是在解決「誰把什麼放進這個箱子」。
