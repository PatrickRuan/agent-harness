AI Agent 白話課
從聊天玩具到會做事的 AI

網路上那堆名詞——prompt engineering、context engineering、harness、MCP、sub-agent、vibe coding—— 其實都在解決同一個問題。這門課用一根主軸把它們串起來。

Model＋ Harness＝ Agent

★這門課的主軸：一個有限的箱子

如果你只能記住一件事，記住這個：

模型沒有記憶，而它的視野是一個有限的箱子。

所有你聽過的縮寫，翻譯過來都是同一個問題的不同答案——誰、在什麼時候、把什麼東西放進這個箱子：

聽起來很潮的名詞翻成白話
Prompt engineering把箱子裡的字寫好
Context engineering決定箱子裡該放什麼
Harness負責裝箱、並反覆呼叫模型的那支程式
Loop engineering設計那個一直重裝箱子的循環
Sub-agent開第二個箱子，別把主箱子弄髒
MCP往箱子裡插工具的標準插頭
Skills說明書需要時才載入

?幾件先說清楚的事

需要程式基礎嗎？

不需要。全部用白話講，程式碼只在少數地方出現，而且都收在「給有程式基礎的同學」的摺疊區裡——不打開完全不影響理解。

頁面上的互動是真的在跑 AI 嗎？

不是。所有互動都是前端模擬，不會連網、不需要金鑰。原因有二：一是不該把 API 金鑰放進公開網頁（這本身就是 M6 會講的安全問題）；二是教學重點在「看懂流程」，模擬就夠清楚了。

資訊有多新？可信嗎？

內容涵蓋到 2026 年 7 月。裡面的日期、人名、產品都經過查證，不是憑印象寫的。

比較新的說法我會誠實標註——例如「harness engineering」和「loop engineering」都是 2026 年才出現的詞，後者寫這頁時只有一個多月大，未必會留下來。學觀念，不要背名詞。

給老師：這是一份靜態網頁教材，零外部相依、可離線開啟、可自由取用修改。 每個模組都能獨立當一節課，建議搭配課堂上實際打開一個 agent 工具邊講邊看。 M6 建議不要省略——學生一旦自己去裝 OpenClaw 這類工具，那一課的內容就是他們的安全帶。

↻這門課是怎麼做出來的

這份教材本身就是一個「人給目標、AI 跑迴圈」的例子——也就是 M3 講的那個迴圈。過程大致是：

  • 先規劃再動工：不是開口就寫程式，而是先確認受眾（大一電機系、不預設程式背景）、決定要不要拆成多頁、要不要加安全章節，寫成計畫、確認之後才建置。
  • 查證勝過印象：AI 的知識有截止日。寫這頁時，「loop engineering」和 Hermes Agent 都在截止日之後才出現——如果照記憶寫，就會寫錯，甚至會理直氣壯地「糾正」正確的說法。所以產品與日期全部另外查過。
  • 在瀏覽器裡真的驗證：每個互動都實際點過、深色模式與手機寬度都看過、主控台確認無錯誤，而不是寫完就當它會動。
  • 純靜態、零相依：整站只有 HTML／CSS／JS，沒有任何外部套件或 CDN，所以可離線、可長期保存、不會因為某個服務關掉就壞掉。
這中間最值得學的一課是第二點：AI 最危險的時候，不是它說「我不知道」，而是它用同樣自信的語氣講一件過期或錯誤的事。 會查證、會要求證據，比會下提示詞重要得多。
